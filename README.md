## 🧭 これは何か（AITL の定義）
**AITL（Architecture for Integrated Technology Logic）** は、  
物理 → デバイス → 制御 → 知能化 を、**一つの因果構造**として設計・説明するためのアーキテクチャです。

断片的な手法の寄せ集めではなく、  
**「どの物理量が、どの状態で、どの制御則に効いて、なぜ性能が出るのか」**を  
**モデル・制御理論・設計判断**の関係として整理します。

---

## 🧠 コアの考え方（役割分担）
このポータルの中心思想は **役割分離**です。

- **PID（内側）**：実時間で安定性と性能を担保する  
- **FSM（外側）**：状態遷移・モード選択・監督（安全側へ倒す）  
- **AI/LLM（最外側）**：故障・劣化・条件変動に対して **再同定／再設計**を支援する  
  - 例：PIDゲイン再同定、FSM遷移ルール修正、モデル更新 など
